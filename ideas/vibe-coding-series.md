# 《Vibe Coding 造一条自动化视频生产线》分期规划

> 围绕本项目真实的架构、工作流、角色与技术选型，做成一个系列；每期 3–10 分钟。
> 组织原则：**边做边播（Dogfooding）**——每期 = 把流水线某个能力做到"能用/更好用"的一个里程碑，同时把这个过程拍成内容。
> 真实进度以 [`content-library/PIPELINE.md`](../content-library/PIPELINE.md) 看板为唯一真相；本文件是脑暴/细化稿。

---

## 主标题

**《Vibe Coding 造一条自动化视频生产线》**

每期标题用**极简直白主词**（视频渲染 / 字幕匹配 / …），"自动化"只留在主标题里，不在每期重复。

> 判断层提醒：纯 "Vibe Coding" 给人"全靠感觉、不读代码"的印象，但本项目有 Schema 校验门 + 角色体系兜底——把"Vibe 也要有护栏"当成卖点，不做真·裸奔。

---

## 三条贯穿全系列的原则

1. **每期一个能力闭环**：一期只讲透一块（渲染 / 字幕 / 音频 / 工作流 / 角色），先把它在仓库里做到可用，再拍。
2. **操作层 + 判断层**：操作层=照着能复现的步骤/配置/命令；判断层=边界与验收、哪一步要人接管、怎么验收。如实但不夸大（反噱头）。
3. **内容即证据**：成片本身就是用这条流水线产出的——以行证言。

## 制作顺序与节奏

**先做 ep02 → ep03 → ep04 → ep05 → ep06 五期"功能期"**，在过程里把工作流、画面与音频质量持续迭代上去；**ep01 总览/总结放到最后制作**——素材最全、故事最清楚时再回头拍开篇（行业里就是"片头最后拍"）。ep01 虽最后做，发布时仍置顶为第 1 期作系列锚点。

1. **ep02 视频渲染**（本期，已在 08 字幕 draft）：打成系列的"画面基线"。
2. **ep03 字幕匹配 → ep04 音频混音**：字幕、音频是后续每期的通用基础设施，先做完后面直接受益。
3. **ep05 工作流构建 → ep06 角色编排**：流程/角色是"上下篇"，做这两期时顺便把半自动（L1）推上去。
4. **ep01 总览/总结**收尾，串起全系列、把技术选型复盘并进来。
- 节奏：每周 1 期、单期投入 ≤8h；完成度 > 完美度，宁可粗糙也不停更。

## 每期展开

### ep02 · 视频渲染（本期）
- **核心问题**：`remotion-composer` 怎么用 `cut.type`/`overlay.type` 把一份配置分发成不同场景。
- **操作层**：演示对比卡 `comparison`、图表 `bar/line/pie/kpi`、终端模拟 `terminal_scene`、截图叠加 `screenshot_scene`、VRM 主持人取景预设。
- **判断层**：**合成录屏的边界**——15–30s 聚焦演示能以假乱真，复杂连续交互会穿帮，必须切真实录屏；数字主持人适合"陪衬"不适合"主角"。
- **同步完善**：补齐常用场景组件；把 VRM 脚部站姿 + 口型纳入质量基线。

### ep03 · 字幕匹配
- **核心问题**：WhisperX 的字级时间戳怎么变成踩点弹跳字幕，并驱动口型。
- **操作层**：真人口播音频 → `transcriber` 出字级 JSON → Remotion 烧录字级高亮/弹跳；中文分词 + 拼音韵母 → VRM 口型。
- **判断层**：ASR 在专业术语/中英混读上的错字率，需要人工校对 + 术语词典；长句对齐漂移怎么修。
- **成片要点**：本期就是 B 轨，字幕效果即"现场证据"。

### ep04 · 音频混音
- **核心问题**：人声与 BGM 如何自动闪避（ducking）、多平台响度如何统一。
- **操作层**：`audio_mixer` 算法级闪避 + 响度标准化；`audio_probe` 找气口对齐转场。
- **判断层**：自动 ducking 的失灵场景（音乐高潮压住人声）、各平台 LUFS 差异、什么时候必须手动调。
- **成片要点**：A/B 波形可视化对比"开/关 ducking"。

### ep05 · 工作流构建
- **核心问题**：怎么把"选题→…→归档"这条流水线变成一台可断点续跑、防漂移的状态机。
- **操作层**：`PIPELINE.md` 看板 + frontmatter 状态；`pipeline_lint.py` 跑 Schema 契约 + 跨阶段防漂移门禁；演示"看板续跑"。
- **判断层**：纯人审 L0.5 的瓶颈；Schema 约束太死 vs 太松；哪些该机器卡、哪些必须人卡。
- **同步完善**：增强 lint 防漂移规则、把 Schema 覆盖到更多阶段。

### ep06 · 角色编排
- **核心问题**：`shared/roles/` 的 Markdown 角色库，怎么从"手动复制 Prompt"走到"多智能体自动流转"。
- **操作层**：角色=system prompt、工作流=user prompt、状态机=调度；用 `pipeline_defs/*.yaml` 串一条可执行管道打样。
- **判断层**：角色膨胀与互相打架、上下文衰减、何时该合并角色；从 IDE 交互到 API 化的真正难点（反馈回路、B 轨需人上传素材的"挂起等待"）。
- **同步完善**：跑通一条角色驱动的半自动管道（迈向 L1）。

### ep01 · 总览（最后制作）
- **核心问题**：把整套生产线一次讲清——为什么把"做视频"变成"编译代码"，这套"IDE 智能体 + Python 工具箱 + React 编译器"三层架构是什么；以及**为什么这么选型**（Remotion vs HyperFrames、AGPL OpenMontage、拒绝数字人对口型 / AI 绘界面）。
- **操作层**：用前 5 期的真实片段串成最小成片 demo——一份配置 → 渲染 → 出片，"改一个参数，画面就变"。
- **判断层**：代码化的代价与适用边界；每个选型的"后悔点"；护城河（反噱头 / 判断层评审门）为什么重要。
- **成片要点**：架构图（`diagram` 平滑变焦）+ 决策树动画；母题金句"视频不是画面堆砌，是参数输入与场景渲染"。
