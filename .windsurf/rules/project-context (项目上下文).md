# ai-ide-workflows 项目上下文

本项目是一个内容创作项目：围绕用 Cursor / Windsurf 等 AI IDE 构建解决现实场景的工作流，制作教程视频与 GitHub 配套资源。

## 核心定位

频道母题：**用 AI IDE 构建工作流，解决现实世界中的各类实际场景问题**。

每期内容尽量同时包含：

1. **操作层**：具体怎么做，包含步骤、Prompt、流程或演示。
2. **判断层**：什么时候该用 / 不该用，如何验收，以及 AI 在哪些点上做不好。

判断层必须如实呈现 AI 的短板，不回避失败点。这是本项目区别于炫技型、软广型 AI 内容的关键。

## 工具定位

- **Cursor**：主线工具，中国用户多，可控性强，步骤清晰可见。
- **Windsurf**：副线 / 对比工具。

关键判断：即使底层模型相同，IDE 设计也会显著影响体验，包括上下文注入、任务规划、错误恢复。

## AI 助手工作原则

- 如果用户的方向明显有问题，要直接提醒，不要顺着错误方向走。
- 内容架构优先于内容运营，先确定故事线再做漏斗优化。
- GitHub 是内容配套资源仓库，不是主产品；建站暂缓，内容优先。
- 详细策略优先参考 `Project Context (项目上下文).md`、`Content Plan (内容计划).md`、`Risk and Moat (风险与护城河).md`。
