## 选题逻辑

每期内容回答一个问题：**这件事，AI IDE 能做吗？做得怎么样？**

项目已精简为两期核心场景验证，按制作 readiness 决定先后顺序。

每期尽量包含两层：
- **操作层**：具体怎么做（步骤、Prompt）
- **判断层**：什么时候该用/不该用、怎么验收结果、AI 在哪些点上做不好

> 判断层的核心支撑是"AI 做不好的部分"。当某个场景存在明显短板时，**必须如实呈现**——这是与"炫技/软广型" AI 内容的根本区别。

## 内容规划（两期核心）

> 标题原则：工具名 + 时效钩子 + 反预期 + 数字/对比，至少满足2条。

| 期数 | 标题（暂定） | 场景 | 判断层核心输出 | 轨道 | 时间预算 |
|------|-------------|------|--------------|------|---------|
| **ep01** | Cursor 做 PPT 实测：能不能替代 Gamma？我用一周验证了验收标准 | 自动化 PPT 制作 | 教会判断"AI 输出质量是否达标、何时需人工接管" | B | 8h |
| **ep02** | 我用代码生成视频：Remotion + AI IDE 让每期后期少花 2 小时 | 自动视频生成 | 教会判断"自动化 ROI 与 Remotion 能力边界" | A | 7h |

> **轨道说明**：A = 概念/总结型，可高度 Remotion 化（代码动画 + 自动字幕，几乎零录屏）；B = 实操演示型，必须真实录屏 + 真人口播，Remotion 只做片头/片尾/转场/字幕/数据卡/多尺寸。

**ep01（PPT 制作）**：用 Cursor / Windsurf 完成真实 PPT 制作任务，记录 AI 在版式、图表、品牌一致性上的偏差，输出可复用的验收清单。

**ep02（自动视频生成）**：展示 Remotion + AI IDE 的完整视频自动化流水线。A 轨可几乎零录屏，本身就是频道母题的终极演示（用 AI IDE 生产 AI IDE 内容）。

## 两轨制作

并非每期视频自动化程度相同，按类型分两条制作轨道，最大化 Remotion 自动化收益、收窄人工断点：

| 轨道 | 适用期数 | 制作方式 | Remotion 角色 | 人工核心 |
|------|---------|---------|--------------|---------|
| **A 轨** | ep02 | 概念/总结型 | 脚本 → Remotion 组件 → 直接渲染成片 | 几乎零录屏（口播可录可 TTS） |
| **B 轨** | ep01 | 实操演示型 | 仅做片头/片尾/转场/字幕/数据卡/多尺寸 | 真实录屏 + 真人口播 + 高光标注 |

**核心判断**：Remotion 不替代"录屏演示 + 真人口播"（可信度来源），而是把人工素材自动组装成片——加字幕、加动画、加转场、出多尺寸。人工只做"录原料 + 标高光"，A 轨甚至可跳过录制。

**工具栈**：Remotion（React 程序化视频）+ Remotion Skills（Windsurf 自然语言生成组件）+ Whisper（字幕时间轴）+ FFmpeg。个人创作者免费，4+ 员工公司需商业授权。

**ep02 是 A 轨试验田**：风险低，且"用代码生成视频"自带话题，用来验证 Remotion 流水线。ep01 为 B 轨，真实录屏验证 PPT 制作能力边界。

## 执行节奏

| 指标 | 目标 |
|------|------|
| 单期总投入 | ≤8小时（录制2h + 后期2h + 配套资源2h + 分发2h） |
| 更新频率 | 每周1期（前5期），之后可调整为10天/期 |
| 断更红线 | 最长不超过2周无更新 |

**优先级**：完成度 > 完美度。宁可粗糙也不停更。

### 渠道匹配判断

| 维度 | 抖音/TikTok 特点 | 本项目内容特点 | 匹配度 |
|------|-----------------|---------------|--------|
| **内容时长** | 15-60秒为主流，3分钟以上完播率骤降 | 需3-8分钟完整呈现"操作层+判断层" | ❌ 不匹配 |
| **受众心智** | 娱乐/消费/情绪刺激，被动滑动 | 工具学习/理性判断，主动搜索 | ❌ 不匹配 |
| **内容深度** | 单点信息，情绪共鸣 | 框架建立，逻辑递进，可复现步骤 | ❌ 不匹配 |
| **小红书差异** | 抖音纯视频流 | 小红书图文卡片可承载步骤截图+文字说明 | ✅ 小红书更适合操作层内容 |

**关键判断**：
- 抖音不是"不做"，而是"暂缓"。先验证 B站+小红书 的内容模型，跑通后再用 Remotion 已生成的 9:16 竖版素材做抖音切片
- 抖音定位：纯引流渠道，不做深度判断层。核心动作是「高光片段+强钩子」吸引点击 B站完整版
- 小红书定位：操作层轻量化呈现（图文卡片），判断层引流到 B站

### 短视频切片策略

抖音/小红书短视频不是长视频的简单截取，而是专门的「钩子产品」。两期内容均可产出切片：

| 切片类型 | 时长 | 内容设计 | 适用平台 |
|---------|------|---------|---------|
| **反预期开场** | 15-30秒 | 先抛一个反直觉结论（"我以为AI能替代Gamma，结果..."） | 抖音、小红书 |
| **高光操作** | 30-60秒 | 最有视觉冲击的自动化过程（Cursor 10秒生成PPT大纲） | 抖音、小红书 |
| **失败瞬间** | 15-30秒 | AI出错片段+人反应（差异化内容，建立信任） | 小红书 |
| **数据冲击** | 10-15秒 | 对比数字（"手动2小时 vs AI 10分钟"） | 抖音 |

**钩子公式（抖音专用）**：
```
[反预期结果] + [制造悬念] + [行动指令]
例："我用Cursor做PPT，结果比Gamma还差。不是AI不行，是我验收标准错了。完整测试过程在置顶链接。"
```

**生产流程**：
```
B站长视频成片（Remotion 9:16竖版已渲染）
  → 人工标注3个高光时间点
  → FFmpeg自动切片（3段）
  → AI生成抖音专用标题+hook文案
  → 分发到抖音/小红书
```

## 自动化发布流程

```
A 轨：脚本 → Remotion 组件（Remotion Skills 生成）→ 渲染成片（16:9 + 9:16）
B 轨：录屏 + 口播 → 导入 Remotion 套模板 → 合成（16:9 + 9:16）
  → Whisper 生成字幕时间轴 → Remotion 自动烧录同步字幕
  → Claude 生成知乎/公众号文章版本（~$0.1/期）
  → Claude 翻译英文字幕 + 文章
  → Claude 生成各平台标题/描述/摘要
  → AI IDE 整理 GitHub 配套文件 + 更新 README
  → 各平台发布
```

目标：A 轨实现"脚本→成片"近全自动；B 轨把后期组装/字幕/多尺寸自动化，人工只录原料 + 标高光。整体 4 小时内从素材到全平台上线。

**注意**：自动化流程本身就是一期内容素材（第9期"用代码生成视频"主题正面展示）。
